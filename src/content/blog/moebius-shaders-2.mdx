## Hatching

The hatching shader is closely based on the technique shown in
[this video by Useless Game Dev](https://www.youtube.com/watch?v=jlKNOirh66E)
(which by the way was also the initial spark for the project as a whole).
The only addition in my version is a secondary outline drawn around cast shadows.

### Hatch texture

All hatching patterns are packed into a single texture.
Each RGB channel holds a different layer: one angle for the lightest shadow zone,
a denser or perpendicular pattern for the next zone, and so on.
Sampling three independent channels from one texture costs the same as sampling one.

### Brightness zones

The shader reads pixel brightness and posterizes it into discrete zones.
Each zone maps to a combination of texture channels.

{/* ILLUSTRATION: brightness gradient -> posterized zones -> corresponding hatch layers (similar to the video diagram) */}

Lit pixels show the base color.
As brightness drops, the shader blends in one channel, then two,
adding lines and cross-hatching progressively.

### Limitation

The texture is sampled in screen space.
Useless Game Dev notes it himself: in an ideal world, the crosshatch pattern would follow the curve of the object it's on.
Here it doesn't: move the camera, and the lines slide across the surface rather than staying anchored to it.
On a still frame this is invisible, but it becomes noticeable on a moving camera.

One approach is to embed the hatching lines into the normal buffer so they read as surface detail.
The problem is that the Scharr filter can't distinguish a geometric crease from a hatch line:
both produce edges, and the mesh ends up covered in unintended outlines.
Resolving this would require a separate buffer or a masking pass (maybe a project for later?).

The playground below illustrates the difference.
Both halves use the same orbiting camera driven by `u_time`.
The top sphere computes the hatch pattern from screen coordinates:
the lines slide across the surface as the camera moves.
The bottom sphere uses the world-space position of the hit point:
the lines stay anchored to the sphere regardless of camera angle.

```glsl live tall
precision mediump float;
uniform vec2 u_resolution;
uniform float u_time;

float sdSphere(vec3 p, float r) { return length(p) - r; }

vec3 calcNormal(vec3 p) {
    vec2 e = vec2(0.001, 0.0);
    return normalize(vec3(
        sdSphere(p + e.xyy, 1.0) - sdSphere(p - e.xyy, 1.0),
        sdSphere(p + e.yxy, 1.0) - sdSphere(p - e.yxy, 1.0),
        sdSphere(p + e.yyx, 1.0) - sdSphere(p - e.yyx, 1.0)
    ));
}

void main() {
    // Split the canvas into two halves, one sphere per half
    float halfH = u_resolution.y * 0.5;
    bool isTop = gl_FragCoord.y >= halfH;
    float PI = 3.14159265;

    // Re-center UV within each half so both spheres appear the same size
    float localY = isTop ? gl_FragCoord.y - halfH : gl_FragCoord.y;
    vec2 uv = (vec2(gl_FragCoord.x, localY) - 0.5 * vec2(u_resolution.x, halfH)) / halfH;

    // Orbit the camera around the sphere using u_time
    float angle = u_time * 0.5;
    vec3 ro = vec3(sin(angle) * 2.5, 0.6, cos(angle) * 2.5);
    vec3 forward = normalize(-ro);
    vec3 right = normalize(cross(vec3(0.0, 1.0, 0.0), forward));
    vec3 up = cross(forward, right);
    vec3 rd = normalize(forward + uv.x * right + uv.y * up);

    // Sphere raymarching
    float t = 0.0;
    bool hit = false;
    for (int i = 0; i < 64; i++) {
        float d = sdSphere(ro + rd * t, 1.0);
        if (d < 0.001) { hit = true; break; }
        t += d;
        if (t > 10.0) break;
    }

    vec3 col = vec3(0.06);
    if (hit) {
        vec3 p = ro + rd * t;
        vec3 n = calcNormal(p);
        vec3 light = normalize(vec3(1.0, 1.5, 1.0));
        float NdotL = dot(n, light);

        // lw is line width: the dark stripe occupies the first lw fraction of each period
        float lw = 0.15;
        float h1 = 1.0;
        float h2 = 1.0;
        float h3 = 1.0;

        if (isTop) {
            // Top: pattern from screen coordinates, slides with the camera
            float scale = 12.0;
            h1 = step(lw, fract((gl_FragCoord.x + gl_FragCoord.y) / scale));
            h2 = step(lw, fract((gl_FragCoord.x - gl_FragCoord.y) / scale));
            h3 = step(lw, fract((gl_FragCoord.x * 2.0 + gl_FragCoord.y) / scale));
        } else {
            // Bottom: pattern from spherical coordinates, anchored to the surface
            float theta = atan(p.z, p.x) / PI; // longitude [-1, 1]
            float phi = acos(clamp(p.y, -1.0, 1.0)) / PI; // colatitude [0, 1]
            float scale = 22.0;
            h1 = step(lw, fract((theta + phi) * scale));
            h2 = step(lw, fract((theta - phi) * scale));
            h3 = step(lw, fract((theta * 2.0 + phi) * scale));
        }

        // Add hatch layers progressively as the surface gets darker
        float hatch = 0.0;
        if (NdotL < 0.65) hatch = max(hatch, 1.0 - h1);
        if (NdotL < 0.30) hatch = max(hatch, 1.0 - h2);
        if (NdotL < 0.05) hatch = max(hatch, 1.0 - h3);

        col = mix(vec3(0.95), vec3(0.05), hatch);
    }

    gl_FragColor = vec4(col, 1.0);
}
```
